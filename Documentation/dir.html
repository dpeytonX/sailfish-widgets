    Detailed Description

    The Dir class

    This class will expose QDir functionality to QML.

    An example usage is as follows.

    Dir {
     id: fileList
     filter: Dir.AllEntries | Dir.Hidden
     sort: Dir.DirsFirst | Dir.Name

     onPathChanged: refresh()
 }

    It does not support file manipulation (creation, reading, writing, delete). This class may be extended in the future as needed.

    Returns the XDG Home directory (usu. /home/nemo). Equivalent to QDir::homePath()

    QString Dir::dirName() const

    Returns the directory name. Unlike QDir::dirName() it will convert the root path to / .

    QStringList Dir::entryList()

    Returns the files in this Dir object's Dir::path(). It provides this object's filter and sort properties as arguments to QDir::entryList().

    This applies the following DirFilter enums (always).

    • QDir::Readable
    • QDir::NoDot

    QQmlListProperty<File> Dir::files()

    Returns the files in this directory as File objects.

    If the internal file list is unpopulated, a call to this method will populate it.

    It will return the same content as entryList().

    See also Dir::entryList().

    void Dir::refresh()

    It will clear the current list and call files(). Afterwords it will emit the filesChanged() method.

    void Dir::setFilter(int filter)

    It set the current filter. Afterwords it will emit the filterChanged() method.

    See also Dir::filterChanged().

    void Dir::setPath(const QString & p)

    It will set the current path to p after running QDir::absolutePath().

    To workaround an issue with indefinate /.. when navigating code, it will force references of /.. to the root directory.

    Afterwords it will emit the pathChanged() method.

    void Dir::setSort(int sort)
